## 3. 상세 테스트 케이스 및 수행 결과

### TC-SHOP-01: 레시피 부족 재료 최저가 장바구니 제안
- **테스트 목적:** 특정 레시피 조회 시 사용자가 보유하지 않은 부족 재료를 정확히 파악하여 묶음 최저가 정보를 반환하는지 확인한다.
- **수행 절차:**
  1. 내 냉장고에 '양파', '마늘'만 있는 상태에서 '돼지고기 김치찌개' 레시피 상세 페이지 진입.
  2. 시스템이 부족 재료(돼지고기, 김치)를 식별하여 `GET /api/v1/shopping/recipes/{recipeId}/missing-items` 호출.
- **기대 결과:** 부족 재료에 대한 쿠팡, 네이버쇼핑 등의 실시간 최저가 목록이 반환되어 화면에 렌더링된다.
- **실제 결과:** 부족 재료 정확히 도출 및 MCP를 통한 쇼핑몰 결과 노출 완료. (**PASS**)

### TC-SHOP-02 & 03: 특정 식재료 최저가 조회 및 가성비 정렬
- **테스트 목적:** 단일 식재료 검색 시 100g/10ml 당 환산 가격(가성비)을 기준으로 상품이 올바르게 정렬되는지 확인한다.
- **수행 절차:**
  1. 쇼핑/검색 화면에서 '닭가슴살 1kg'을 키워드로 최저가 조회.
  2. 반환된 상품 리스트의 `price`, `unit_price` 확인.
- **기대 결과:** 단순히 총액(`price`)이 낮은 순서가 아니라, 100g 당 환산 가격(`unit_price_per_100`)이 가장 저렴한 항목이 최상단에 노출된다.
- **실제 결과:** 총액은 15,000원이지만 100g당 1,000원인 A상품이 총액 12,000원이지만 100g당 1,200원인 B상품보다 상단에 정상 노출됨. (**PASS**)

### TC-SHOP-04: AI(GraphRAG) 기반 대체재 추천
- **테스트 목적:** 현재 시장 가격이 비싸거나 품절된 식재료에 대해 합리적인 대체재와 사유를 제공하는지 확인한다.
- **사전 조건:** '시금치'의 가격이 급등한 상황 가정 (테스트 데이터 셋업).
- **수행 절차:**
  1. '시금치'가 포함된 레시피 상세 보기 또는 쇼핑 탭 진입.
  2. `GET /api/v1/shopping/alternatives` 호출.
- **기대 결과:** "현재 시금치 가격이 평년 대비 40% 상승했습니다. 유사한 조리법의 청경채를 추천하며, 영양성분이 85% 일치합니다."와 같은 대체 사유(`reason`) 및 절감 가능한 비용(`price_advantage`)이 함께 렌더링된다.
- **실제 결과:** FastAPI의 `/internal/v1/ai/graph-reasoning`에서 반환된 대체재 목록과 추천 사유가 UI에 정상 표시됨. (**PASS**)

### TC-SHOP-05: 외부 쇼핑몰 딥링크 이동 (Purchase URL)
- **테스트 목적:** 제안된 상품을 클릭했을 때 실제 외부 쇼핑몰 상품 페이지로 정확히 리다이렉트되는지 확인한다.
- **수행 절차:**
  1. 조회된 쇼핑 리스트 중 쿠팡/네이버 상품 1개의 [구매하기] 버튼 클릭.
- **기대 결과:** 새 탭(`target="_blank"`)이 열리며 전달받은 `purchase_url`로 정상 이동한다.
- **실제 결과:** 링크 파손 없이 정확한 상품 상세 페이지 딥링크로 접속됨. (**PASS**)

### TC-SHOP-06: 쇼핑 데이터 캐싱 동작 (부하 방지)
- **테스트 목적:** 빈번하게 조회되는 식재료의 외부 API(MCP) 반복 호출을 방지하고 DB 캐시를 활용하는지 검증한다.
- **수행 절차:**
  1. A 사용자가 '양파' 최저가 검색 (API 호출됨).
  2. 1분 뒤 B 사용자가 동일하게 '양파' 최저가 검색.
- **기대 결과:** B 사용자의 요청 시 내부적으로 MCP 외부 API를 타지 않고, DB의 `shopping_item_mcp` 테이블에 저장된 캐시 데이터(TTL 이내)를 반환하여 응답 속도가 현저히 빠르다. (`is_mcp_source: false` 반환 확인)
- **실제 결과:** 2번째 요청 시 캐시 히트(Cache Hit) 확인, 응답속도 50ms 이내로 단축됨. (**PASS**)

- **특이사항:** AI 및 쇼핑 연동 로직 특성 상, FastAPI 분석 서버(project_final_backend2)의 응답 지연(Timeout)이 간헐적으로 발생할 수 있음.
- **조치사항:** 프론트엔드에서 데이터를 불러오는 동안 사용자 이탈을 막기 위해 스켈레톤 UI(Skeleton UI) 및 "최저가 정보를 실시간으로 수집 중입니다..." 같은 로딩 메시지를 추가 적용 완료함.
